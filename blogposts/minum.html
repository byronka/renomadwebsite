A minimalistic Java web framework

November 4, 2023

I have published a stable Java web framework focusing on minimalism.

https://github.com/byronka/minum

You will be hard-pressed to find a modern web framework as single-mindedly focused on minimalism, simplicity, and ease-of-use.

The first question I get asked about this is: "why would I use this? What differentiates it from the many other web frameworks available?".

The answer is: Probably, it is not for you. This is a framework intended for the rare obsessive starving for minimalism in a baroque world. Conventional frameworks, like Spring Boot, put high guardrails around capabilities. They embrace a paradigm of building blocks (e.g. "Transactional"). Most of the whirring parts are safely hidden away.

This requires a large amount of code, which increases the complexity. New concepts require reading copious documentation on each building block. It also makes maintenance harder, since stacktraces will be made nearly useless by the hidden machinery, and a great deal of understanding will be obscured in the name of ease of use.

In contrast, the Minum project follows the self-imposed constraint that smaller and lighter is better. Being an advocate for software quality, I built a system replete with automated testing, documentation, code invariants, and examples.

The goals of this software are to provide excellence in:

  • Minimalism
  • Ease of use
  • Maintainability
  • Debuggability
  • Performance
  • Documentation
  • Testing

It requires Java 21, for its virtual threads.
